\section{Cloud Deployment}
\subsection{Cloud Deployment models}
\begin{itemize}
    \item Private cloud: uses vituralization for on demand resources and can be located off-premises, direct control over resources available, and how data is stored
    \item Public cloud: internet accessible for general public, no access to data center and hardware
    \item Hybrid cloud: mix, for seasonal spikes or cloud bursting
    \item Multicloud : combine different CSP
    \item Community cloud: shared by specific community with common interest
\end{itemize}

\subsection{Regulations on the cloud}
Different region have different view on how data should be managed
\begin{itemize}
    \item General Data Protection Regulation (GDPR): EU, explicit consent, data breaches notification, encrypted, anonymized PI, can't leave EU borders unless guarantee same level of protection, fine of 20 mil or 4\% of worldwide revenue whichever is greater
    \item \url{https://commission.europa.eu/law/law-topic/data-protection/data-protection-explained_en}
    \item Brazil's Lei Geral de Proteçao de Dados (LGPD)
    \item California's Consumer Privacy Act (CCPA)
    \item USA' Health Insurance Portability and Accountability Act (HIPPA)
    \item Japan's Act on Protection of Personal Information
    \item Thailand Personal Data Protection Act (PDPA)
    \item Canada's Personal Information and Electronic Documents Act (PIPEDA)
\end{itemize}

\subsection{Cloud computing roles}
\begin{itemize}
    \item Data scientist: run computationally expensive analysis on cloud
    \item Machine Learning scientist: train and deploy ML models on the cloud
    \item Data engineer: build pipelines on the cloud to ingest, transform, and store big data
    \item Data analyst: access data on the cloud via BI tools
    \item Cloud architect: solution architect for the cloud
    \item Cloud engineer: role for anything technical on cloud
    \item DevOps engineer: Software \textbf{Dev}elopment + IT \textbf{Op}erations, IaC
    \item Security engineer: test and assess security of data on cloud
\end{itemize}
