\section{Cloud Deployment}
\subsection{Cloud Deployment models}
\begin{itemize}
    \item Private cloud: uses vituralization for on demand resources and can be located off-premises, direct control over resources available, and how data is stored
    \item Public cloud: internet accessible for general public, no access to data center and hardware
    \item Hybrid cloud: mix, for seasonal spikes or cloud bursting
    \item Multicloud : combine different CSP
    \item Community cloud: shared by specific community with common interest
\end{itemize}

\subsection{Regulations on the cloud}
Different region have different view on how data should be managed
\begin{itemize}
    \item General Data Protection Regulation (GDPR): EU, explicit consent, data breaches notification, encrypted, anonymized PI, can't leave EU borders unless guarantee same level of protection, fine of 20 mil or 4\% of worldwide revenue whichever is greater
    \item \url{https://commission.europa.eu/law/law-topic/data-protection/data-protection-explained_en}
    \item Brazil's Lei Geral de Proteçao de Dados (LGPD)
    \item California's Consumer Privacy Act (CCPA)
    \item USA' Health Insurance Portability and Accountability Act (HIPPA)
    \item Japan's Act on Protection of Personal Information
    \item Thailand Personal Data Protection Act (PDPA)
    \item Canada's Personal Information and Electronic Documents Act (PIPEDA)
\end{itemize}

\subsection{Cloud computing roles}
\begin{itemize}
    \item Data scientist: run computationally expensive analysis on cloud
    \item Machine Learning scientist: train and deploy ML models on the cloud
    \item Data engineer: build pipelines on the cloud to ingest, transform, and store big data
    \item Data analyst: access data on the cloud via BI tools
    \item Cloud architect: solution architect for the cloud
    \item Cloud engineer: role for anything technical on cloud
    \item DevOps engineer: Software \textbf{Dev}elopment + IT \textbf{Op}erations, IaC
    \item Security engineer: test and assess security of data on cloud
\end{itemize}

\section{Cloud Providers and Case Studies}
\subsection{An overview of providers}
\begin{itemize}
    \item AWS, Azure, GCP, Alibaba Cloud, salesforce, IBM Cloud, Oracle cloud, tencent cloud
    \item Best cloud provider meets company needs
\end{itemize}

\subsection{Amazon Web Services}
\begin{itemize}
    \item launched in 2006
    \item more than 175 services in computing, storage, analytics, security and ML
    \item \textgreater 31\% market share
    \item EC2 for computation, RDS for database, S3 for storage
    \item Data Services: Kinesis for real time data and analytics, Redshift for analytics data warehousing, Sagemaker for predictive analytics
    \item Case Study: NerdWallet takes too long to deploy ML model, use Sagemaker to reduce training times to days and training cost by 75\%
\end{itemize}

\subsection{Microsoft Azure}
\begin{itemize}
    \item launched in 2008
    \item Integration with Microsoft products
    \item \textgreater 24\% market shared
    \item Azure Blob storage for file storage, Azure Virtual Machine for computation, Azure SQL Database for databases, 
    \item Data Services: Microsoft Fabric is SaaS for enterprise use that brings together different Microsoft solutions which covers everything from data movement to data science, realtime analytics, and BI, Data Lake storage for storing data before cleaning, Stream Analytics for real-time analytics and Machine Learning, to train and deploy ML models
    \item Azure customers: SIEMENS, ĽORÉAL, THE WORLD BANK
    \item Case Study: Ottawa Hospital needs cost effective and secure disaster recovery solution, Microsoft IaaS, Azure storage for medical imaging data, Azure Site Recovery for recovery processes automatically when a data loss incident occurs saving 50\% on disaster recovery cost and compliant infra
\end{itemize}

\subsection{Google Cloud}
\begin{itemize}
    \item launched in 2010
    \item Google Cloud Anthos for hybrid multi-cloud solution in 2019
    \item \textgreater 11\%
    \item Google Cloud Storage, Google Cloud Compute Engine, Google cloud SQL
    \item Data Services: BigQuery for data warehouse, Data flow for batch and stream data processing, AutoML for ML model training and development
    \item Google Cloud Customers : HSBC, PayPal, vodafone
    \item Case Study: LUSH needs availability and stability during peak loads migrated entire global infra to Google cloud, Google cloud Compute Engine to test and provision in migrating services with data in Google Cloud SQL reducing 40\% in hosting cost
    \item \url{cloud.google.com/customer/lush/}
\end{itemize}
